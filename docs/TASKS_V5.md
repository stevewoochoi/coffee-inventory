# TASKS_V5.md - 매장 직원 관점 UX 고도화 (발주 · 재고 · 일상 운영)

> 매장 직원이 매일 쓰는 화면을 **실제 업무 흐름** 기준으로 재설계
> 핵심: "생각하지 않아도 되는 발주", "한눈에 보이는 재고"

---

## 매장 직원의 하루 시뮬레이션

```
07:00  출근 → 대시보드 확인 (오늘 뭐 해야 하지?)
07:05  저재고 알림 확인 → 발주 필요 항목 체크
07:10  발주 작성 (원두 2종 + 우유 + 컵) → 공급사별 자동 분류 → 발주 전송
09:00  어제 발주한 물건 도착 → 입고 확인
12:00  점심 피크 후 재고 빠르게 체크
15:00  유통기한 임박 우유 폐기 처리
17:00  내일 발주할 거 있나 한번 더 확인
```

**현재 문제점:**
- 발주 시 공급사 ID를 직접 입력해야 함 → 누가 ID를 외우나?
- 뭘 발주해야 하는지 직접 판단해야 함 → 시스템이 알려줘야 함
- 이전 발주 내역 참고가 안 됨 → 매번 처음부터 작성
- 부재료 → 공급사 → 포장단위 연결이 직관적이지 않음

## PHASE 17: 스마트 발주 시스템

### TASK-054 | 발주 필요 항목 자동 추천 API

- [ ] GET /api/v1/ordering/needs?storeId={id}
  - 발주 필요 판단 기준:
    - 현재 재고 ≤ 안전재고 (min_stock_qty) → **긴급**
    - 현재 재고 ≤ 안전재고 × 1.5 → **권장**
    - 최근 7일 평균 소비량 기준 3일 이내 소진 예상 → **예측**
  - Response:
    ```json

          "itemId": 1,
          "itemName": "에티오피아 예가체프 원두",

          "currentStock": 2.5,
          "unit": "kg",
          "minStock": 5.0,
          "avgDailyUsage": 1.2,
          "daysUntilEmpty": 2.1,
          "lastOrderDate": "2026-02-20",
          "lastOrderQty": 10,
          "suppliers": [
            {
              "supplierId": 3,
              "supplierName": "원두상사",
              "packagings": [
                { "packagingId": 7, "label": "1kg × 1봉", "unitPrice": 25000 },
                { "packagingId": 8, "label": "5kg × 1봉", "unitPrice": 110000 }
              ],
              "leadTimeDays": 2
            }
          ],
          "suggestedQty": 3,
          "suggestedPackagingId": 7
        }
      ],
      "recommended": [...],
      "predicted": [...]
    }
    ```

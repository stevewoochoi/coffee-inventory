## 🐛 BUG #1. 통화 단위 불일치 (JPY → KRW)

**페이지**: 매장 계정 > 수주관리 > 발주

### 현상
- 본사 계정에서 브랜드에 상품 배정 시 가격을 **엔(JPY)** 으로 설정함
- 매장 계정에서 발주를 넣을 때 동일 상품의 가격이 **원(KRW)** 으로 표시됨

### 기대 동작
- 본사에서 설정한 통화(JPY)가 매장 계정 발주 화면에도 동일하게 표시되어야 함
- 통화 단위는 브랜드/상품 설정 값을 그대로 따라야 함

### 확인 필요 사항
- [ ] 상품/브랜드 통화 필드가 DB에 저장되고 있는지
- [ ] 매장 발주 화면에서 통화 필드를 조회/렌더링하고 있는지
- [ ] 하드코딩된 `₩` 또는 `KRW` 표시가 있는지 검색

## 🐛 BUG #2. 매장 계정에서 "발송" 처리 가능 (권한 분리 필요)

**페이지**: 매장 계정 > 수주관리 > 발주현황 > 발주내역 상세

### 현상
- 발주 상세 페이지 하단에 **[발송]** 버튼이 노출됨 (스크린샷 빨간 원 표시)
- 매장 계정에서 [발송] 버튼 클릭 시 "발송되었습니다" 토스트가 뜨고 상태가 `발송됨`으로 변경됨

### 기대 동작
- **발송 처리는 본사(공급사) 계정의 권한**이어야 함
- 매장 계정에서는 발송 버튼이 노출되지 않거나, 노출되더라도 비활성화되어야 함
- 또는 매장 계정에서는 [발송] 버튼 자체를 렌더링하지 않도록 권한 체크 필요

### 확인 필요 사항
- [ ] 프론트엔드: 사용자 role(STORE / HQ / SUPPLIER)에 따른 버튼 조건부 렌더링
- [ ] 백엔드: 발송 API에 권한 체크(403) 추가 (프론트만 막으면 안 됨)

## 🐛 BUG #3. PDF 다운로드 실패

- 실제 PDF 파일이 다운로드되지 않음

### 확인 필요 사항

- [ ] 백엔드 PDF 생성 라이브러리(예: iText, Apache PDFBox 등) 정상 동작 여부
- [ ] 한글 폰트 누락으로 인한 렌더링 실패 가능성
- [ ] AWS Lightsail 서버 환경에서 PDF 생성에 필요한 시스템 폰트/라이브러리 설치 여부
- [ ] CORS / 인증 헤더 누락 가능성

## 🐛 BUG #4. "확정" 상태에서 입고 버튼이 동작함

**페이지**: 매장 계정 > 수주관리 > 입고

### 현상
- 입고 대기 발주 목록에서 발주 상태가 **`확정`** 인데도 [입고] 버튼이 노출되고 클릭 가능함
- 클릭 시 입고 상세 페이지로 이동됨

### 기대 동작
- [입고] 버튼은 **`발송됨`** 상태에서만 노출되거나 활성화되어야 함
- `확정` 상태에서는 버튼이 미노출 또는 비활성화(disabled) 상태여야 함

### 참고
- 현재 화면(image_1.png)에서 `#9 메가커피`(확정), `#10 냅킨코리아`(확정) 모두 [입고] 버튼이 활성화되어 있음
- `#8 발트로스트원두`(발송됨)와 동일하게 노출되고 있어 사용자가 혼동함

### 수정 방향
```
if (order.status === 'SHIPPED') {
  show 입고 button (active)
} else {
  hide 입고 button OR show as disabled
}
```

## 🐛 BUG #5. 부분입고 처리 UX 및 완료 판정 로직

**페이지**: 매장 계정 > 수주관리 > 입고

### 현상
1. 부분입고 처리를 했지만, 어떤 상품을 입고했는지 화면에서 식별 불가
2. 부분입고 처리 후 다시 입고 페이지에 들어가 **남은 상품을 입고 처리**해도 여전히 `부분입고` 상태에 머무름
   - 현재 로직: 2개 상품 모두를 동시에 입고 처리해야 발주 건이 `완료` 처리됨
3. `입고 #1`, `#2`, `#3`, `#4` 항목 우측에 단순히 `COMPLETED`만 표시되어 어떤 상품이 입고되었는지 알 수 없음

### 기대 동작

#### (A) 입고 처리 화면에서 이미 입고된 상품 식별
- 부분입고 후 다시 진입 시, 이미 입고된 상품은 **"이미 입고됨"** 으로 표시
- 미입고 상품만 수량 입력/입고 처리 가능하도록 노출

#### (B) 완료 판정 로직 수정
- 현재: 한 번에 모든 상품을 입고 처리해야 `완료`
- 수정 후: **이미 입고된 상품 + 이번에 입고 처리한 상품 = 발주 전체 수량**이면 자동으로 `완료` 상태로 전환
- 즉, 누적 입고 수량 기준으로 완료 판정해야 함

#### (C) 입고 이력 직관성 개선
- 현재: `입고 #1   공급사 #9   COMPLETED`
- 변경: `입고 #1   공급사 #9   요거트 파우더 1kg x1   COMPLETED`
- 입고 항목별로 어떤 상품을 몇 개 입고했는지 한눈에 보이도록 표시

### 확인 필요 사항
- [ ] 백엔드: 발주 라인 아이템(line item)별 입고 누적 수량 추적 필드
- [ ] 백엔드: 모든 라인 아이템 누적 입고 수량 == 발주 수량일 때 `COMPLETED`로 자동 전환
- [ ] 프론트엔드: 입고 화면에서 라인 아이템별 입고 상태(`미입고` / `부분입고` / `완료`) 표시
- [ ] 프론트엔드: 입고 이력 카드에 입고된 상품 정보 노출

## 🐛 BUG #6. 입고 처리 후 재고 미반영

**페이지**: 매장 계정 > 재고관리 > 재고현황

### 현상
- 발주 상품을 정상적으로 입고 처리 완료(`COMPLETED`)했음
- 재고관리 > 재고현황 페이지에 진입하면 **"재고 데이터 없음"** 으로 표시됨
- 부족 재고만 필터가 켜져 있을 가능성도 있으나, 전체 재고도 비어 있는지 확인 필요

### 기대 동작
- 입고 처리가 완료되면 매장의 현재 재고에 즉시 반영되어야 함
- `재고관리 > 재고현황` 에서 입고된 상품(요거트 파우더 1kg, 율무차 파우더 900g 등)이 조회되어야 함

### 확인 필요 사항
- [ ] 입고 처리 트랜잭션에서 재고 테이블(stock/inventory) UPDATE 또는 INSERT가 실행되는지
- [ ] 입고 → 재고 반영 이벤트/리스너 구현 여부 (Spring Application Event 또는 동기 호출)
- [ ] 매장(Store) 단위 재고 조회 쿼리에서 store_id / brand_id 필터가 정확한지
- [ ] "부족 재고만" 토글이 기본 ON 상태인지, OFF 시에는 데이터가 보이는지 확인

### 의심되는 원인
1. 입고 완료 시 재고 INSERT/UPDATE 로직 누락
2. 부분입고 상태에 머물러 있어 실제로는 재고 반영 트리거가 발화되지 않았을 가능성 (BUG #5 와 연관)
3. 재고 조회 시 매장-브랜드 매핑 조건 오류
